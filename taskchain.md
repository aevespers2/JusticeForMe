# Task Chain

## Product directive

JusticeForMe provides read-only Linux host observations and local evidence review. Its immediate objective is to preserve the current collector/dashboard scope while defining the contracts, evidence semantics, accessibility evidence, and governance required for safe integration with the Repository `0`/`1` portable device-security foundation.

## Current priority

**P0 — Documentation, evidence semantics, exact-artifact accessibility review, and integration-boundary review.**

No collector remediation, remote transport, credential use, scheduled enforcement, accessibility certification, Pages publication, or production authority is authorized.

## Success criteria

- Current collector behavior and non-capabilities are documented accurately.
- Report schema `1.0`, supporting artifacts, thresholds, and failure semantics are explicit.
- GitHub Pages exposes a safe project and architecture guide without publishing private audit evidence.
- Accessibility review is tied to exact source and rendered artifacts without representing automated checks as certification.
- The Repository `0` observation boundary and Repository `1` baseline/capability boundary are proposed without activation.
- Material gluing obstructions and required shared fixtures are recorded.
- Release, punch-list, changelog, validation, privacy, accessibility, rollback, and recovery guidance agree.

## Builder rules

Builders may work only on `READY` tasks. Documentation, local validation, synthetic fixtures, accessibility protocols, and evidence-preserving contract proposals are permitted. Changes that inspect additional sensitive surfaces, modify the host, transmit reports, use credentials, create remote services, issue capabilities, certify accessibility, publish Pages, or remediate configuration require separately approved tasks.

## Evidence rules

Every material task records:

- exact source commit and changed files;
- exact rendered artifact and digest when visual or accessibility claims are made;
- implemented versus proposed behavior;
- commands, tools, browsers, viewports, input methods, and assistive-technology versions;
- positive, negative, partial, failed, blocked, unknown, corrected, withdrawn, and superseded results as applicable;
- privacy and evidence-handling effects;
- unsupported platforms, browsers, assistive technologies, or checks;
- limitations and residual risks;
- correction, rollback, incident, and supersession paths;
- approval and authority status.
