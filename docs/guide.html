What this project does

JusticeForMe reads Linux privilege and configuration metadata from an authorized local host, writes a timestamped report directory, and lets a reviewer load the generated report.json into a dependency-free browser dashboard.

What it does not do

The project does not prove compromise, attribute activity, determine intent, certify compliance or accessibility, upload reports, deploy a service, or replace a forensic examiner. Dashboard thresholds prioritize review; they are not universal security baselines.

Safe review sequence

  1. Confirm authorization and preserve the original report directory.
  2. Verify REPORT-SHA256SUMS.txt.
  3. Open the dashboard from a trusted local checkout.
  4. Select report.json; the browser processes it locally.
  5. Compare indicators with trusted package metadata, known-good baselines, and documented changes.
  6. Record uncertainty, incomplete collection, and every transformation or copy.

Accessibility review boundary

Current status is DOCUMENTED_NOT_CERTIFIED. Automated validation checks structure, local routes, local assets, and source identity. Manual keyboard, screen-reader, contrast, zoom, reflow, comprehension, and error-recovery review must be recorded separately for one exact non-deploying artifact.

Privacy and publication boundary

Real reports can reveal hostnames, users, groups, local paths, privileged binaries, package changes, and persistence mechanisms. Do not publish reports or attach them to public issues. GitHub Pages publication is not authorized; the repository workflow validates pull-request source only and has read-only permissions.

Interpretation rule

A finding is an indicator for investigation, not proof of misconduct. Preserve raw artifacts, document chain of custody, and seek specialist review when conclusions carry operational, employment, legal, or public consequences.
